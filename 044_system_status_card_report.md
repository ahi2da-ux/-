# 044 시스템 상태 카드 추가 리포트

## 작업 목표

로그인 문제와 DB 연결 문제를 분리해서 볼 수 있도록 MY 탭에 시스템 상태 카드를 추가했습니다.

## 왜 필요한가

현재 짠테크맵의 가장 큰 병목은 Supabase Auth QA 계정 로그인입니다.
하지만 로그인 실패가 있다고 해서 앱 전체가 실패한 것은 아닙니다.

운영 관점에서는 아래 항목을 따로 봐야 합니다.

- Supabase 설정이 앱에 들어갔는지
- 장소 데이터가 Supabase DB에서 오는지, 목업 fallback인지
- 로그인 상태인지
- 게스트 체험 기록이 남아 있는지
- 제보/포인트/1억 챌린지 데이터가 불러와졌는지

## 변경한 내용

### 1. MY 탭에 시스템 상태 카드 추가

표시 항목:

- Supabase 설정
- Auth 상태
- 장소 데이터 출처
- DB 메시지
- 게스트 체험 기록 수
- 내 제보/포인트
- 1억 챌린지 상태

### 2. 장소 데이터 새로고침 버튼 추가

카드 오른쪽 상단의 새로고침 버튼을 누르면 장소 데이터를 다시 불러옵니다.

### 3. 폴백 상태 안내

장소 데이터가 목업 fallback이면 출시 전에는 Supabase DB 상태로 바뀌어야 한다는 안내를 보여줍니다.

## 수정 파일

- `ContentView.swift`

## QA 기준

- 앱 빌드 성공
- MY 탭에서 시스템 상태 카드 표시
- 새로고침 버튼으로 장소 데이터 재조회 가능
- Supabase DB/목업 fallback 상태를 명확히 확인 가능
