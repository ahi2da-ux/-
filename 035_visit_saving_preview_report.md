# 035 방문 완료 체험 모드 리포트

## 목적

Supabase 테스트 계정이 아직 로그인되지 않아도 사용자가 장소 상세에서 `방문 완료` 흐름을 확인할 수 있게 했습니다.

## 변경 내용

### 1. 비로그인 방문 완료 동작 변경

기존:

- 비로그인 상태에서 `방문 완료` 버튼을 누르면 토스트만 표시
- 사용자는 절약액 기록 화면을 볼 수 없음

변경:

- 비로그인 상태에서 `방문 완료` 버튼을 누르면 `방문 완료 체험` 시트가 열림
- 메뉴 선택 가능
- 일반 기준가, 짠테크맵 이용가, 예상 절약액 확인 가능
- 절약액 직접 입력 가능

### 2. 실제 저장과 체험 저장 분리

체험 모드는 DB에 저장하지 않습니다.

실제 저장은 여전히 로그인된 Supabase 세션이 있을 때만 가능합니다.

## 사용자 관점 효과

로그인 전에도 사용자는 아래를 이해할 수 있습니다.

1. 내가 이 장소를 이용하면 얼마나 절약되는지
2. 1억 챌린지에 어떤 금액이 쌓이는지
3. 메뉴별 가격 차이가 어떻게 계산되는지
4. 로그인하면 무엇이 실제 저장되는지

## 운영 관점 판단

이 기능은 가입 전 전환율을 높이는 데 도움이 됩니다.
로그인 장벽을 완전히 없애지는 않지만, 사용자가 기능 가치를 먼저 확인할 수 있게 합니다.

## 남은 작업

테스트 계정 Confirm 후 실제 저장 루프를 검증해야 합니다.

1. 로그인
2. 장소 상세
3. 방문 완료
4. 절약 기록 저장
5. MY 카드 반영
6. 최근 기록 취소
