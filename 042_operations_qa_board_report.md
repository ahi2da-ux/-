# 042 운영 QA 보드 추가 리포트

## 작업 목표

짠테크맵 MVP가 기능 구현 단계에서 운영 검증 단계로 넘어갈 수 있도록, MY 탭에 운영 QA 체크 보드를 추가했습니다.

## 왜 필요한가

현재 앱은 지도, 제보, 포인트, 알림, 1억 챌린지 기능이 각각 많이 구현되어 있습니다.
하지만 출시 전에는 개별 기능보다 더 중요한 것이 있습니다.

사용자가 실제로 누르는 순서대로 데이터가 끝까지 이어지는지 검수해야 합니다.

예:

1. 로그인
2. 장소 상세
3. 방문 완료
4. 가격 제보
5. 운영자 승인
6. 포인트 지급
7. 알림 생성
8. 1억 챌린지 절약액 반영

## 변경한 내용

### 1. MY 탭에 운영 QA 보드 추가

MY 탭의 1억 챌린지 테스트 카드 아래에 운영 QA 보드를 추가했습니다.

체크 항목:

1. 로그인/세션 복원
2. 방문 완료 → 1억 챌린지
3. 가격 제보 + 사진 업로드
4. 운영자 승인/반려
5. 포인트/알림/챌린지 연결

### 2. 체크 상태 저장

체크 상태는 `@AppStorage`를 사용해 시뮬레이터 기기에 저장됩니다.
앱을 껐다 켜도 체크 상태가 유지됩니다.

### 3. 로그인 전 보호

로그인하지 않은 상태에서는 체크 항목을 누를 수 없게 했습니다.
대신 “먼저 Supabase QA 계정 로그인을 성공시킨 뒤 체크를 진행해주세요.”라는 안내를 보여줍니다.

## 수정 파일

- `ContentView.swift`

## QA 기준

- 앱 빌드 성공
- MY 탭에서 운영 QA 보드 표시
- 로그인 전에는 체크 비활성화
- 로그인 후 체크 가능
- 초기화 버튼으로 체크 상태 리셋 가능

## 다음 단계
