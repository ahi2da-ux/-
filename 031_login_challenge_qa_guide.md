# 031 로그인 기반 1억 챌린지 QA 가이드

이 문서는 시뮬레이터에서 1억 챌린지 기능을 직접 확인하기 위한 테스트 절차입니다.

## 1. 테스트 계정 만들기

1. Supabase 대시보드에 접속합니다.
2. Authentication > Users 메뉴로 이동합니다.
3. Add user를 누릅니다.
4. 아래 값으로 테스트 유저를 만듭니다.
   - Email: `jjantechmap.qa@gmail.com`
   - Password: 별도 보관한 QA 비밀번호
5. 이메일 인증이 켜져 있다면 해당 유저를 Confirm 처리합니다.

주의: Supabase Auth는 `example.com` 같은 예약 도메인을 거부할 수 있어서, QA 샘플 이메일은 실제 형식의 도메인으로 맞췄습니다.

## 2. 앱에서 로그인하기

1. 시뮬레이터에서 짠테크맵을 실행합니다.
2. 하단 MY 탭으로 이동합니다.
3. 로그인 카드에서 `QA 샘플값 채우기`를 누릅니다.
4. 이미 Supabase에 유저를 만들어두었다면 `로그인으로`를 누릅니다.
5. 로그인 버튼을 누릅니다.
6. 상단 프로필 영역이 로그인 상태로 바뀌면 성공입니다.

## 3. 1억 챌린지 핵심 루프 확인

1. 탐색 또는 지도 탭에서 장소를 하나 선택합니다.
2. 장소 상세 하단의 `방문 완료` 버튼을 누릅니다.
3. 메뉴를 선택하고 절약 기록을 저장합니다.
4. MY 탭으로 돌아옵니다.
5. 1억 챌린지 카드의 누적 절약액이 증가했는지 확인합니다.
6. 1억 챌린지 상세 화면에서 최근 절약 기록이 보이는지 확인합니다.

## 4. 중복 방지 확인

1. 같은 장소와 같은 메뉴를 다시 방문 완료 처리합니다.
2. 2시간 중복 제한 메시지가 나오면 정상입니다.
3. 앱이 종료되거나 화면이 멈추면 실패입니다.

## 5. 기록 취소 확인

1. 1억 챌린지 상세 화면의 최근 절약 기록에서 취소 버튼을 누릅니다.
2. 확인창에서 취소하기를 선택합니다.
3. 누적 절약액이 줄어들고 해당 기록이 사라지면 정상입니다.

## 6. 현재 판단

- 로그인 입력 UX: 개선 완료
- Supabase Auth 실제 로그인: 테스트 계정 생성 후 확인 필요
- 1억 챌린지 저장 루프: 로그인 후 실사용 QA 필요
- 다음 단계: 실제 계정으로 시뮬레이터에서 저장/취소까지 검증
