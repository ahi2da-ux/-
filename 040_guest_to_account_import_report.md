# 040 게스트 체험 기록 실제 계정 이전 작업 리포트

## 작업 목표

Supabase 로그인 계정이 준비되면, 로그인 전에 저장한 1억 챌린지 체험 기록을 실제 사용자 장부로 옮길 수 있게 했습니다.

## 왜 필요한가

현재 QA 계정 로그인이 아직 `invalid_credentials`로 막혀 있습니다.
하지만 사용자가 앱을 체험하다가 나중에 로그인했을 때 기록이 끊기면 제품 경험이 나빠집니다.

그래서 이번 단계에서는 로그인 전 체험 기록을 기기에 저장하고, 로그인 후 사용자가 직접 실제 장부로 옮길 수 있는 연결 장치를 만들었습니다.

## 변경한 내용

### 1. 게스트 체험 기록에 장소/메뉴 ID 저장

기존에는 체험 기록에 장소명과 절약액만 저장했습니다.
이제는 실제 Supabase RPC `log_saving`에 보낼 수 있도록 아래 값을 함께 저장합니다.

- `placeID`
- `menuID`

### 2. MY 탭 체험 장부 카드에 실제 이전 버튼 추가

로그인 상태에서 체험 기록이 남아 있으면 MY 탭 체험 장부 카드에 `실제 장부로` 버튼이 표시됩니다.

버튼을 누르면:

1. 기기에 저장된 체험 기록을 읽습니다.
2. 각 기록을 `log_saving` RPC로 Supabase에 저장합니다.
3. 성공한 기록은 기기에서 제거합니다.
4. 실패한 기록은 다시 시도할 수 있게 남겨둡니다.
5. 1억 챌린지 요약 데이터를 다시 불러옵니다.

### 3. 데이터 보호 방식

이전 중 실패한 기록은 삭제하지 않습니다.
네트워크 오류나 중복 기록 제한이 걸려도 사용자의 체험 데이터가 사라지지 않게 했습니다.
