# PRICE_REPORT_POLICY — 가격 제보 운영 정책

> **문서 버전:** v1.0-beta  
> **작성일:** 2026-05-09  
> **적용 범위:** 짠테크맵 베타 테스트 기간 전체  
> **담당:** Ops Agent / 관리자

---

## 1. 가격 제보 승인 기준

가격 제보가 승인되려면 아래 **모든 조건**을 충족해야 합니다.

| # | 조건 | 설명 |
|---|------|------|
| 1 | **장소 실존** | 네이버 지도·카카오맵에서 확인 가능한 실제 영업 중인 장소여야 합니다. |
| 2 | **가격 범위 합리성** | 해당 카테고리 평균 대비 ±50% 이내여야 합니다. (예: 커피 평균 4,500원 → 2,250원~6,750원 범위) |
| 3 | **증빙 자료 첨부** | 사진/영수증/메뉴판 중 **최소 1장** 이상 첨부해야 합니다. (§3 참조) |
| 4 | **제보일 기준 7일 이내 촬영** | 증빙 사진의 촬영 시점이 제보일로부터 7일 이내여야 합니다. |
| 5 | **메뉴/품목 명시** | "커피 아메리카노 2,000원"처럼 구체적인 메뉴명과 가격이 명시되어야 합니다. |
| 6 | **중복 제보 아님** | 동일 장소·동일 메뉴의 미검수 제보가 이미 존재하지 않아야 합니다. (§4 참조) |

### 승인 후 처리
- 제보 상태를 `approved`로 변경합니다.
- 제보자에게 **포인트 10P**를 지급합니다.
- 해당 장소의 대표 가격 정보를 업데이트합니다.
- 제보 승인 알림을 발송합니다.

## 2. 가격 제보 반려 기준

아래 조건 중 **하나라도** 해당되면 반려합니다.

| # | 반려 사유 | 조치 |
|---|-----------|------|
| 1 | **증빙 없음** | 사진/영수증 미첨부 → 반려 후 "증빙을 첨부해주세요" 안내 |
| 2 | **가격 비현실적** | 카테고리 평균 대비 50% 초과 차이 → 반려 후 "가격을 다시 확인해주세요" 안내 |
| 3 | **장소 미확인** | 지도에서 확인 불가 또는 폐업 장소 → 반려 후 "장소를 확인할 수 없습니다" 안내 |
| 4 | **메뉴 불명확** | "음료 1,000원"처럼 구체적이지 않은 경우 → 반려 후 "메뉴명을 구체적으로 작성해주세요" 안내 |
| 5 | **허위/조작 의심** | 사진과 장소 불일치, 가격 변조 흔적 → **즉시 반려 + 경고 1회** |
| 6 | **광고/홍보 목적** | 특정 업소 홍보 의도가 명확한 경우 → 반려 후 "홍보 목적 제보는 불가합니다" 안내 |
| 7 | **오래된 정보** | 증빙 촬영일이 제보일로부터 7일 초과 → 반려 후 "최신 정보로 다시 제보해주세요" 안내 |

### 허위 제보 누적 처리
| 경고 횟수 | 처분 |
|-----------|------|
| 1회 | 경고 알림 발송 |
| 2회 | 제보 기능 7일 정지 |
| 3회 | 제보 기능 영구 정지 + 베타 테스터 자격 박탈 검토 |

## 3. 사진/영수증/메뉴판 기준

### 3.1 인정되는 증빙 유형

| 유형 | 요구사항 | 예시 |
|------|----------|------|
| **메뉴판 사진** | 장소명 또는 간판이 함께 보이면 가장 좋음. 메뉴명+가격이 선명하게 읽혀야 함 | 카페 벽면 메뉴판, 테이블 메뉴판 |
| **영수증 사진** | 상호명, 메뉴명, 금액, 결제일이 확인 가능해야 함 | POS 영수증, 카드 전표 |
| **앱 주문 캡처** | 배달앱·키오스크 화면의 메뉴명+가격이 보여야 함 | 배민, 쿠팡이츠 주문 화면 |
| **현장 촬영 사진** | 가격표·POP·현수막 등 가격이 표시된 매장 내외부 사진 | 할인 안내 현수막, 가격표 |

### 3.2 인정되지 않는 증빙

- ❌ 텍스트만 입력하고 사진 미첨부
- ❌ 블로그/SNS에서 퍼온 타인 촬영 사진
- ❌ 가격이 읽히지 않을 정도로 흐린 사진
- ❌ 촬영일 확인 불가 + 오래된 것으로 의심되는 사진
- ❌ 포토샵/편집 흔적이 있는 사진

### 3.3 사진 품질 최소 요건
- **해상도:** 640×480 이상
- **가독성:** 메뉴명과 가격이 육안으로 읽혀야 함
- **위변조:** EXIF 메타데이터 조작 의심 시 반려

## 4. 중복 제보 처리

### 4.1 중복 판정 기준

아래 **3가지 조건이 모두 일치**하면 중복으로 판정합니다:

1. **동일 장소** (place_id 또는 위치 반경 50m 이내)
2. **동일 메뉴/품목** (메뉴명 유사도 80% 이상)
3. **기존 승인된 가격과 동일** (±5% 이내)

### 4.2 중복 제보 시 처리 방식

| 상황 | 처리 |
|------|------|
| 기존 승인 가격과 **동일** (±5% 이내) | 중복으로 반려, "이미 등록된 가격입니다" 안내 |
| 기존 승인 가격과 **다름** (5% 초과 차이) | **가격 변동 제보**로 처리 → 새로 검수 진행 |
| 기존 제보가 **미검수 대기 중** | 후발 제보는 대기열 뒤로 배치, 선발 제보 우선 처리 |

### 4.3 가격 변동 제보 보상
- 기존 가격 대비 실제로 변동이 확인된 경우: **포인트 15P** (일반 제보 대비 50% 추가)
- 변동이 확인되지 않은 경우: 반려 처리, 포인트 미지급

## 5. 오래된 가격 처리

### 5.1 가격 유효 기간

| 카테고리 | 유효 기간 | 근거 |
|----------|-----------|------|
| **식음료 (카페, 식당)** | 90일 | 계절 메뉴·물가 변동 반영 |
| **생활 서비스 (세탁, 미용)** | 180일 | 상대적으로 가격 변동 적음 |
| **마트/편의점** | 30일 | 할인·프로모션 빈도 높음 |
| **기타** | 90일 (기본값) | — |

### 5.2 만료 처리 프로세스

```
[유효기간 30일 전] → 장소 카드에 "⚠️ 가격 확인 필요" 배지 표시
[유효기간 도달]   → 가격 상태를 "미확인(stale)"으로 변경
[유효기간 + 30일] → 해당 가격 정보를 지도에서 비활성화 (삭제는 아님)
```

### 5.3 가격 갱신 유도
- 만료 임박 장소를 "가격 업데이트 필요 🔄" 목록으로 별도 노출
- 해당 장소 가격을 갱신 제보한 사용자에게 **포인트 10P** 지급

## 6. 관리자 검수 체크리스트

관리자는 매 제보 검수 시 아래 체크리스트를 순서대로 확인합니다.

### ✅ 검수 체크리스트

```
□ 1. 장소 실존 확인
     → 네이버 지도 또는 카카오맵에서 검색하여 영업 중인지 확인
     → 폐업/이전 확인 시 즉시 반려

□ 2. 증빙 사진 확인
     → 사진이 1장 이상 첨부되었는지 확인
     → 메뉴명과 가격이 선명하게 읽히는지 확인
     → 편집/조작 흔적이 없는지 확인

□ 3. 가격 합리성 검증
     → 카테고리 평균 대비 ±50% 범위 내인지 확인
     → 극단적으로 저렴한 경우 프로모션/이벤트 여부 확인

□ 4. 중복 여부 확인
     → 동일 장소+동일 메뉴의 기존 승인 제보 검색
     → 중복이면 반려, 가격 변동이면 변동 제보로 처리

□ 5. 제보 내용 정확성
     → 메뉴명이 구체적으로 작성되었는지 확인
     → 장소 위치(핀)가 실제 위치와 일치하는지 확인

□ 6. 최종 판정
     → 승인: 포인트 지급 + 가격 정보 업데이트
     → 반려: 사유 선택 + 안내 메시지 발송
     → 보류: 추가 확인 필요 시 "보류" 처리 (24시간 내 재검수)
```

### 검수 소요 시간 목표
- **일반 제보:** 접수 후 24시간 이내 검수 완료
- **긴급 제보 (허위 신고 등):** 접수 후 6시간 이내 처리
- **베타 기간 중:** 하루 2회 (오전 10시, 오후 6시) 일괄 검수
